Hamster Simulator Documentation

This documentation provides an overview of how the Hamster Simulator works, including key concepts, classes, and methods.

Key Concepts

  • Hamster Class

    Represents the hamster's position, direction, and movement in the simulation.

  • Constants

    Includes predefined values such as colors (BLAU, BLUE, RED) and directions (NORD, OST).

  • Constructors

    Various constructors for initializing the hamster with parameters such as position, direction, and color.

  • Initialization

    The hamster is initialized using the init() method or constructors, ensuring valid position and direction.

  • Movement Methods

    Controls hamster's movements, including vor() for moving forward and linksUm() for turning left.

  • Utility Methods

    Methods like gib(), nimm(), and kornDa() manage the hamster's grain-related actions.

  • Getter Methods

    Methods like getReihe(), getSpalte(), and getAnzahlKoerner() return the hamster's state, including its position and grain count.

  • Exceptions

    Methods may throw exceptions like HamsterNichtInitialisiertException or MauerDaException for invalid actions.

  • Cloning and Comparison

    The hamster can be cloned with clone() and compared for equality with equals().

  • Standard Hamster

    The getStandardHamster() method provides the default hamster configuration for the simulation.

Sample Code


// Hamster Simulator Targetting Mechanism Example
void main() {
    Hamster hamster = new Hamster(0, 0, Hamster.OST, 0); 

    int targetX = 3;
    int targetY = 12;

    while (hamster.getReihe() != targetX) {
        if (hamster.getReihe() < targetX) {
            faceDirection(hamster, Hamster.SUED);
        } else {
            faceDirection(hamster, Hamster.NORD);  
        }

        if (hamster.vornFrei()) {
            hamster.vor();
        }
    }
    while (hamster.getSpalte() != targetY) {
        if (hamster.getSpalte() < targetY) {
            faceDirection(hamster, Hamster.OST);
        } else {
            faceDirection(hamster, Hamster.WEST);  
        }

        if (hamster.vornFrei()) {
            hamster.vor();
        }
    }
}

void faceDirection(Hamster hamster, int direction) {
    while (hamster.getBlickrichtung() != direction) {
        hamster.linksUm();  
    }
}

Python Documentation

This documentation outlines the key concepts of Python programming, focusing on core features of the language used in various projects.

Key Concepts

  • Data Types

    Python supports a variety of built-in data types, including integers, floats, strings, lists, tuples, sets, and dictionaries. Each type has its own set of operations and uses in programming.

  • Variables and Constants

    Variables store values in memory, while constants are used to store values that should not be changed throughout the program. Python does not have true constant variables, but naming conventions (e.g., all uppercase) can be used.

  • Control Flow

    Control flow structures like `if`, `else`, `elif`, `while`, and `for` allow the program to execute code conditionally or repeatedly, enabling the creation of complex logic and loops.

  • Functions

    Functions are blocks of reusable code that perform a specific task. They are defined using the `def` keyword and can accept parameters and return values.

  • Classes and Objects

    Python supports object-oriented programming, which allows you to define classes that model real-world entities. Classes can contain methods and attributes, and objects are instances of these classes.

  • Exceptions and Error Handling

    Python uses a try-except block to handle runtime errors. This allows the program to continue executing even when an error occurs, without crashing.

  • List Comprehension

    List comprehension provides a concise way to create lists by performing an operation on each item in an existing iterable, allowing for more readable and efficient code.

  • Iterators and Generators

    Iterators allow you to loop through items in a collection, while generators are a type of iterator that yield items one at a time, which is memory efficient when working with large data sets.

  • Modules and Packages

    Modules are individual Python files that contain reusable code, while packages are collections of modules. Python allows you to import and use modules and packages to organize code efficiently.

  • Lambda Functions

    Lambda functions are anonymous functions defined with the `lambda` keyword. They are useful for simple operations and can be passed as arguments to other functions.

  • Decorators

    Decorators are functions that modify the behavior of other functions or methods. They are commonly used for logging, access control, or caching results in Python.

  • File I/O

    Python provides built-in functions for reading from and writing to files. The `open()` function is used to access files, and data can be read or written in various formats like text, binary, and CSV.

Python Libraries Documentation

This section outlines the key Python libraries that are frequently used in various data analysis, machine learning, and scientific computing projects.

Key Libraries

  • NumPy

    NumPy is a fundamental library for numerical computing in Python. It provides support for large multi-dimensional arrays and matrices, along with a collection of mathematical functions to operate on these arrays.

  • Pandas

    Pandas is a powerful library used for data manipulation and analysis. It provides data structures like DataFrames and Series, which allow for efficient handling and analysis of structured data, such as time series and tabular data.

  • Matplotlib

    Matplotlib is a plotting library used for creating static, animated, and interactive visualizations in Python. It is widely used for generating plots, charts, and graphs for data analysis and visualization.
